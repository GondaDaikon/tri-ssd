# スキル開発ルール

このルールは `skills/` ディレクトリ内の `SKILL.md` ファイルを編集する際に適用されます。

## フロントマター必須フィールド

```yaml
---
name: スキル名（必須）
description: >
  スキルの説明（必須）。
  トリガーワードを含める。
---
```

### descriptionの要件

descriptionには以下を必ず含める:

1. **スキルが起動する条件**
2. **ユーザーが使用するキーワード/トリガーワード**
3. **典型的なユースケース**

## 必須セクション

### 1. Instructions（必須）

スキルが呼び出されたときの動作手順を明記:

```markdown
## Instructions

1. ユーザーの入力を解析
2. 適切なコマンドを選択
3. コマンドを実行
4. 結果を報告
```

### 2. 利用可能なコマンド（必須）

```markdown
## 利用可能なコマンド

| コマンド | 用途 |
|---------|------|
| /xxx | 説明 |
```

### 3. Examples（推奨）

使用例を具体的に記載。

### 4. Limitations（推奨）

制限事項を明記。

## レイヤースキップ禁止

スキル内で以下を明記すること:

```markdown
## Limitations

- L1なしでL2を生成しない
- L2なしでL3を生成しない
- doc_status: draft のL3からコード生成しない
```

## ファイルサイズ制限

| 推奨 | 最大 |
|------|------|
| 300-500行 | 1000行 |

## 段階的開示

1. 基本機能を先に案内
2. オプション機能は必要時に説明
3. デフォルト動作を明確に
